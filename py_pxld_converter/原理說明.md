# Python 版 PXLD v3 工具 - 原理說明

這份文件說明 Python 版轉檔器/分析器的資料流與核心規則，目標是與 repo 內既有 C++ 版保持一致。

---

## 目標與資料流

### 轉檔（FSEQ → PXLD v3）

1. 讀取 FSEQ V2（uncompressed）header，取得：
   - `channel_data_offset`：frame 資料起點
   - `channel_count`：每個 frame 的 channel bytes
   - `frame_count`、`step_time_ms`（換算 fps）
2. 讀取 `mapping_config.json`：
   - 每個 slave 的 channel 範圍（`channels.start/count`）
   - 每個 slave 的 outputs（LED 型別、LED 數、每顆占用幾個 channels、output 起始 channel）
3. 對每個 frame：
   - 依 mapping 把各 slave 的 channel data 擷取出來
   - 把各 slave 的 channel data 轉成 PXLD v3 統一 RGBW（4 bytes/LED）
   - 寫入 PXLD v3 frame（含 frame header + slave table + pixel data）
4. 關檔時回填：
   - `total_frames`
   - `file_crc32`

### 分析（PXLD → 暗幀區段）

1. 讀取 PXLD v3 header
2. 逐 frame 讀出每個 slave 的 pixel bytes
3. 判斷「整個 frame 是否全黑」，並找出連續暗幀區段

## Mapping 載入與 v3 data_offset/data_length 重算

mapping 內 output 的 `data_offset/data_length` 在 v3 會被自動重算：

- `data_length = output.count * 4`
- `data_offset` 為同一個 slave 的 outputs 依序累加

對應 Python： [load_mapping_config](file:///c:/Users/bl91920/Documents/code/git/LightShow_xLight/py_pxld_converter/mapping.py)

這代表 v3 的 Pixel Data 佈局只取決於每個 output 的 LED 數量與排列順序，而不依賴 config 內寫死的 offset/length。

## Channel 擷取（1-indexed → 0-indexed）

xLights 的 channel 編號是 1 起算；frame data 在檔案中是 byte array（0 起算）。

擷取時以：

- `array_start = slave.channels.start - 1`
- 複製 `slave.channels.count` bytes（若不足就截斷或補 0）

對應 Python： [extract_slave_channels](file:///c:/Users/bl91920/Documents/code/git/LightShow_xLight/py_pxld_converter/channel_extractor.py)

## 轉換成 v3 統一 RGBW（4 bytes/LED）

### RGB LED（APA102C / WS2812B）

FSEQ 來源為 `[R,G,B]`，輸出 v3 為 `[R,G,B,0xFF]`：

- R/G/B 可套用兩個處理：
  - `cap-white`：當 R/G/B 都 ≥ threshold 時，把三者設為 threshold
  - `brightness`：全域亮度百分比縮放：`out = (value * brightness_percent) // 100`
- W 固定為 `0xFF`

### 單色 PWM（STANDARD_LED）

FSEQ 來源為 `[PWM]`，輸出 v3 為 `[0,0,0,PWM]`：

- W 不做 brightness 縮放（保持與 C++ 行為一致）

對應 Python： [convert_slave_data](file:///c:/Users/bl91920/Documents/code/git/LightShow_xLight/py_pxld_converter/channel_extractor.py)

## PXLD v3 寫檔格式（little-endian）

### File Header（64 bytes）

Python 以 `struct` 寫入固定大小 header，核心欄位：

- Magic `"PXLD"`
- `major_version=3`, `minor_version=0`
- `fps`（整數）與 `fps_milli`（fps*1000）
- `total_frames` 先寫 0，關檔時回填
- `file_crc32` 先寫 0，關檔時計算後回填
- `checksum_type=1` 表示 CRC32

對應 Python： [PxldWriter](file:///c:/Users/bl91920/Documents/code/git/LightShow_xLight/py_pxld_converter/pxld.py)

### 每個 Frame

- Frame Header（32 bytes）：`frame_id`、`slave_table_size`、`pixel_data_size`
- Slave Table（N * 24 bytes）：每個 slave 一筆 entry，含 `slave_id/channel_start/channel_count/pixel_count/data_offset/data_length`
- Pixel Data：依照 slave entry 的 `data_offset/data_length` 切片

---

## CRC32（從 offset 27 到 EOF）

CRC32 規則：

- 算法：IEEE CRC32（Python 使用 `zlib.crc32`）
- 範圍：從 **offset 27**（`checksum_type`）到檔尾
- 回填：寫回 file header 的 **offset 23-26**

對應 Python： [PxldWriter._update_crc32](file:///c:/Users/bl91920/Documents/code/git/LightShow_xLight/py_pxld_converter/pxld.py)

## 暗幀判定

對每顆 pixel（4 bytes）：

- 若任一 pixel 的 `(R,G,B)` 有非 0 → 不是暗幀
- W 的判定採寬鬆規則：`w` 只允許 `0` 或 `0xFF`（因為 RGB LED 的 W 固定為 0xFF、PWM LED 的 W 可能為 0）

對應 Python： [_is_frame_all_zero](file:///c:/Users/bl91920/Documents/code/git/LightShow_xLight/py_pxld_converter/cli_pxld_analyzer.py)
