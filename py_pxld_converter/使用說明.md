# Python 版 PXLD v3 工具 - 使用說明

此目錄提供兩個 Python CLI：

- `fseq_to_pxld_py`：FSEQ V2（uncompressed）→ PXLD v3
- `pxld_analyzer_py`：PXLD v3 暗幀（全黑）區段分析

工具完全使用 Python 標準函式庫（不需額外安裝套件）。

---

## 快速開始

在 repo 根目錄執行：

### 1) 轉檔：FSEQ → PXLD v3

```bash
python -m py_pxld_converter.cli_fseq_to_pxld <input.fseq> <mapping_config.json> <output.pxld>
```

常用參數：

```bash
python -m py_pxld_converter.cli_fseq_to_pxld input.fseq pxld_converter/mapping_config.json output.pxld --brightness 100 --cap-white 0 --lenient
```

參數說明：

- `--brightness 0-100`：全域亮度縮放（預設 100）
- `--cap-white 0-255`：白光上限（當 R/G/B 都 ≥ threshold 時，把 R/G/B 設為 threshold；0 表示停用）
- `--lenient`：遇到 mapping/範圍錯誤時以 0 補齊並繼續（預設 strict，遇到問題直接中止）

### 2) 分析：找暗幀（全黑）連續區段

```bash
python -m py_pxld_converter.cli_pxld_analyzer <input.pxld> [MIN_FRAMES]
```

例如，列出至少連續 10 幀的暗幀區段：

```bash
python -m py_pxld_converter.cli_pxld_analyzer output.pxld 10
```

## mapping_config.json 怎麼用

轉檔會依照 `mapping_config.json` 的 slave/channel 設定，把每個 frame 的 channel data 分段擷取出來，並轉成 PXLD v3 統一的 RGBW（每顆 LED 固定 4 bytes）。

repo 內既有的 mapping 檔在：

- [mapping_config.json](file:///c:/Users/bl91920/Documents/code/git/LightShow_xLight/pxld_converter/mapping_config.json)

注意：

- config 裡每個 output 的 `data_offset/data_length` 在 v3 會被程式自動重算（每顆 LED 固定 4 bytes），所以不需要手動調這兩個欄位。

## 驗證方式

### 1) 跑自測

會自動產生最小 FSEQ/mapping，轉出 PXLD 後再讀回比對 payload：

```bash
python -m py_pxld_converter.selftest
```

### 2) 用既有 mapping 做載入檢查

例如快速看 slave 數、channels、pixels、以及某個 slave 的 v3 資料長度是否符合預期（主燈箱 2448 bytes、外框 top/bottom 6848 bytes 等）：

```bash
python -c "from py_pxld_converter.mapping import load_mapping_config; m=load_mapping_config('pxld_converter/mapping_config.json'); print(len(m.slaves), m.settings.total_channels, m.settings.total_pixels, m.slaves[-1].total_data_length)"
```

## 已知限制與相容性

- 僅支援 **FSEQ V2 uncompressed**（與 C++ 版一致）。若 FSEQ header 顯示 compression_type != 0 會直接拒絕。
- PXLD v3 的 slave entry 內 `channel_start/channel_count` 欄位是 `uint16`：若 mapping 的 channel 起點超過 65535，寫入 PXLD 時會依既有實作截斷到 16-bit，以維持與目前 C++ 版輸出相同的行為。
